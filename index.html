Write-Ups Database

This application is a database for a retail chain that initially stored all write-ups on paper inside filing cabinets. It allows for easier tracking of who has been written up for that year, and how many write-ups they have received. I wrote this in 2017.

The main page is grouped by employee, then category of write-up. The count for each type is displayed. Records are revealed after clicking the category. Employees can be filtered by 'Active' (still employed with the company), 'Inactive' (no longer employed with the company), or 'All' (both). By default, records are shown for the present year, but the user can switch to a previous year.
Upper management can see all employees, while middle can only see the ones they manage (I'm using a key combo to log out).
The user is able to add new employees from the Write-Ups Database into the Employee Database.
Employees can be edited from here as well.
When adding a write-up, lists of employees and managers are populated from the Employee Database. Signatures can be drawn with a mouse or touchscreen, and are stored as Base32 strings inside the database.
Records can be edited easily as well.
Employees can be searched within both the records section and the edit employee section.
The user is also able to navigate their history.
Fruity Pebbles: The Game

I was always curious how games worked, and had an idea for a simple and absurd one. So in 2014, I read an HTML5 game design book, and developed a very inefficient Fruity Pebbles: The Game. I rewrote it in 2016, and brought it to ZapCon, a local arcade convention. After that, it appeared on a podcast, YouTube, and even a top games of 2016 list.

Fruity Pebbles: The Game adds new content to a 1989 Fruity Pebbles commercial. Fred Flintstone's adventure begins with the original ad. It's important to the plot, so it can't be skipped.
Fred learns important tips for surviving this brutally difficult game in the tutorial.
In the first stage, Fred must face off against multiplying Barnies.
In the second, Fred goes up against the Master Rapper, who turns out to be Barney.
Finally, Fred must utilize all the skills he's learned in order to defeat the final boss.
With MegaBarney defeated, and Fred's pebbles in their rightful stomach, peace is restored to Bedrock.
Employee Database

I wrote this in 2015 for a retail chain. It stores employee information and pictures.

Employees can be filtered by 'Active', 'Inactive', 'All', or 'No Photo' (to see whose picture needs to be taken).
When adding an employee, the picture can be browsed for, or dragged and dropped. It can also be uploaded later from a smartphone's camera.
Employee info can also be edited.
Users can search the database.
This application has a history panel, too.
